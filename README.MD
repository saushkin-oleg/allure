# Allure TestOps Exporter

Инструмент для экспорта тест-кейсов и тест-планов из Allure TestOps в формате PDF с поддержкой кириллицы, структурированным отображением шагов, предусловий и ожидаемых результатов.

## Возможности

- 📤 Экспорт отдельных тест-кейсов в PDF
- 📚 Экспорт целых тест-планов с группировкой по разделам
- 🔍 Автоматическое определение разделов по префиксам в названиях (NTPR-XX)
- 📊 Детальная информация: шаги, предусловия, ожидаемые результаты, теги, метаданные
- 📎 Поддержка вложений, комментариев и истории изменений
- 🔧 Отладочный режим с сохранением сырых JSON данных
- 🌐 Полная поддержка кириллицы (Arial/Helvetica)
- 🎨 Чистое форматирование: отступы, маркированные списки, иерархия шагов

## Установка

1. **Клонируйте репозиторий:**
```bash
git clone <repository-url>
cd allure-testops-exporter
```
Установите зависимости:

```
pip install requests reportlab
```
Создайте файл конфигурации:

```bash
# Запустите скрипт для создания шаблона
python main.py
```
Или создайте config.json вручную:
```
json
{
    "api_token": "your_api_token_here",
    "allure_url": "https://allure-testops.office.it-bastion.com",
    "output_dir": "./exports",
    "testplan_id": 39,
    "testcase_id": 4448,
    "project_id": 3,
    "filter_prefix": "NTPR"
}
```

## Использование
Основной скрипт с CLI
```bash
# Экспорт тест-плана и одного тест-кейса (режим по умолчанию)
python main.py

# Экспорт только тест-плана
python main.py --mode testplan

# Экспорт только одного тест-кейса
python main.py --mode single

# С указанием конкретного файла конфигурации
python main.py --config my_config.json

# Переопределение параметров из командной строки
python main.py --testplan-id 42 --testcase-id 123 --output-dir ./my_exports

# Без сохранения сырых JSON
python main.py --no-raw

# С включением сырых данных в PDF
python main.py --include-raw-pdf
```
Отдельные скрипты
Экспорт одного тест-кейса (отладка):

```bash
python debug_testcase.py
```
Экспорт всего тест-плана:

```bash
python export_testplan.py
```
### Параметры командной строки
Параметр	Описание
--config, -c	Путь к файлу конфигурации (по умолчанию: config.json)
--mode, -m	Режим экспорта: testplan, single, both (по умолчанию: both)
--testplan-id	ID тест-плана (переопределяет конфиг)
--testcase-id	ID тест-кейса (переопределяет конфиг)
--project-id	ID проекта (переопределяет конфиг)
--output-dir	Директория для сохранения (переопределяет конфиг)
--no-raw	Не сохранять сырые JSON
--include-raw-pdf	Включать сырые данные в PDF
Формат PDF
Для отдельного тест-кейса:
Заголовок с ID и датой генерации

Название тест-кейса

Мета-информация (даты создания/изменения, автор, теги)

## Описание

Предусловия (с маркированным списком)

Сценарий с шагами (иерархическая структура)

Ожидаемые результаты для каждого шага

Комментарии и история изменений

Для тест-плана:
Статистика (количество тест-кейсов и разделов)

Группировка по разделам (NTPR-01, NTPR-02 и т.д.)

Для каждого тест-кейса: название, ID, мета-информация, предусловия, шаги, ожидаемые результаты

Настройка шрифтов
По умолчанию используется:

Arial (Windows) - полная поддержка кириллицы

Helvetica (Linux/macOS) - без кириллицы

Для Linux/macOS рекомендуется установить шрифт с кириллицей:

```bash
# Ubuntu/Debian
apt-get install fonts-liberation

# Или укажите путь к шрифту в коде:
pdf_gen = PDFGenerator(font_path="/usr/share/fonts/truetype/liberation/LiberationSans-Regular.ttf")
```

## Особенности
Группировка по разделам: тест-кейсы автоматически группируются по префиксам в квадратных скобках (например, [NTPR-01-01])

Иерархия шагов: правильно отображаются вложенные шаги и ожидаемые результаты

Очистка HTML: все HTML-теги удаляются, текст форматируется в читаемый вид

Отладка: сохраняются сырые JSON для анализа проблем

## Устранение неполадок
Проблемы с кириллицей
Если в PDF отображаются квадратики вместо русских букв:

Убедитесь, что установлен Arial (Windows) или другой шрифт с кириллицей

Для Linux установите fonts-liberation или ttf-mscorefonts-installer

## Ошибки API
Проверьте правильность API токена в config.json

Убедитесь, что URL Allure доступен из вашей сети

Проверьте права доступа к проекту и тест-кейсам

Нет тест-кейсов в PDF
Проверьте параметр filter_prefix - возможно, тест-кейсы имеют другой префикс

Убедитесь, что в тест-плане есть тест-кейсы с указанным ID

Разработка и доработка
Добавление новых полей в PDF
Найдите соответствующий метод в pdf_generator.py

Добавьте новый блок с использованием стилей

При необходимости добавьте новые стили в _init_styles()

Изменение форматирования
Стили находятся в методе _init_styles() класса PDFGenerator

Отступы регулируются параметром leftIndent в стилях

Маркированные списки формируются в _format_step()
